  <action>
1. Install prerequisites if not present:
   - Rust nightly toolchain: `rustup toolchain install nightly`
   - WASM target: `rustup +nightly target add wasm32-unknown-unknown`
   - rust-src component: `rustup +nightly component add rust-src`
   - wasm-pack: Install from pre-built binary (NOT cargo install -- it's archived and may fail to compile). Use: `curl https://rustwasm.github.io/wasm-pack/installer/init.sh -sSf | sh` or download from https://github.com/rustwasm/wasm-pack/releases/tag/v0.14.0
   - FRB codegen: `cargo install flutter_rust_bridge_codegen`

2. Scaffold the project using FRB's create command from the project root:
   ```
   flutter_rust_bridge_codegen create --name pulijoodam
   ```
   IMPORTANT: This command scaffolds INTO an existing directory. If it requires an empty directory, create a temp directory, scaffold there, then move files into project root. Preserve existing files (.planning/, SPEC.md, game-description.md, game-rules.md, board-graph.md, .git/).

   If `create` fails or conflicts with existing files, use the manual approach:
   - `flutter create --project-name pulijoodam .` (or a temp dir)
   - `flutter_rust_bridge_codegen integrate` to add Rust integration
   - This two-step approach is documented in FRB docs as an alternative.

3. Edit `rust/Cargo.toml` to add console_error_panic_hook:
   ```toml
   [dependencies]
   flutter_rust_bridge = "2"
   console_error_panic_hook = "0.1"
   ```
   Ensure crate-type includes `["cdylib", "staticlib"]`.

4. Replace `rust/src/api/simple.rs` with the Phase 1 API:
   ```rust
   /// Initialize the WASM module. Call once at app startup.
   pub fn init_app() {
       console_error_panic_hook::set_once();
   }

   /// Trivial function to prove the Rust-WASM-Dart pipeline works.
   pub fn greet(name: String) -> Result<String, String> {
       if name.is_empty() {
           return Err("Name cannot be empty".to_string());
       }
       Ok(format!("Hello from Rust WASM, {}!", name))
   }

   /// Deliberately trigger a panic to verify console_error_panic_hook works.
   /// For testing/verification only.
   pub fn test_panic() {
       panic!("This is a test panic to verify console_error_panic_hook is working");
   }
   ```

   CRITICAL: Zero unwrap policy at FFI boundary. `greet` returns `Result<String, String>`, NOT `String`. `test_panic` is intentionally panicking for verification purposes only.

5. Ensure `rust/src/lib.rs` has the correct module structure that FRB expects (typically `mod api;` with `pub mod simple;` inside `api/mod.rs`). Follow whatever structure FRB's scaffold created -- only modify the function bodies, not the module layout.

6. Run FRB codegen to generate Dart bindings:
   ```
   flutter_rust_bridge_codegen generate
   ```
   This creates/updates files in `lib/src/rust/` including `frb_generated.dart` and `api/simple.dart`.
