  <action>
    Create src/hooks/useDrag.ts implementing SVG drag with Pointer Events API per research Pattern 3.

    The hook signature:
    ```
    useDrag(opts: {
      svgRef: RefObject<SVGSVGElement>;
      board: (Piece | null)[];
      phase: Phase;
      currentTurn: Role;
      legalMoves: LegalMove[];
      onNodeTap: (id: number) => void;
      disabled: boolean;
    }): {
      isDragging: boolean;
      dragPieceId: number | null;
      dragPos: { x: number; y: number } | null;
      handlers: {
        onPointerDown: (nodeId: number, e: React.PointerEvent) => void;
        onPointerMove: (e: React.PointerEvent) => void;
        onPointerUp: (e: React.PointerEvent) => void;
      };
    }
    ```

    Key implementation details:
    - DRAG_THRESHOLD = 5 SVG units. Below this, treat as tap (delegate to onNodeTap).
    - Convert screen coords to SVG using `svg.getScreenCTM()!.inverse()` with `DOMPoint.matrixTransform()`.
    - Call `setPointerCapture(e.pointerId)` in pointerDown to prevent lost-drag on fast movement.
    - Drag only activates for pieces on board (movement phase for any piece; during chain-hop for the chaining tiger).
    - Goat placement stays tap-only: if phase === 'placement' and currentTurn === 'goat' and the node is empty, do NOT enter drag.
    - On drop: find nearest valid node within SNAP_RADIUS (30 SVG units). If valid, call onNodeTap(fromNode) then onNodeTap(toNode) to reuse existing move logic. If invalid, return cancel (piece snaps back via CSS transition).
    - Export `findNearestValidNode` for unit testing.

    Update Board.tsx:
    - Add a ref to the SVG element (`svgRef`).
    - Accept new optional prop `lastEvents?: GameEvent[]` (for ScreenReaderAnnouncer, added in Task 2).
    - Call useDrag hook with appropriate params.
    - In the pieces layer, for the dragged piece (dragPieceId), override x/y with dragPos. Apply transform: `scale(1.2)` and `filter: drop-shadow(0 2px 4px rgba(0,0,0,0.4))` while dragging.
    - Attach onPointerDown to piece elements (currently have pointerEvents: 'none'). Change piece pointerEvents to 'auto' for movement-phase pieces. Keep pointerEvents: 'none' during animations.
    - Attach onPointerMove and onPointerUp to the SVG element itself (not individual pieces) so drag tracking works even when cursor leaves the piece.
    - When a piece node has pointerDown handler, also add `touch-action: none` CSS to prevent browser scroll/zoom during drag.
