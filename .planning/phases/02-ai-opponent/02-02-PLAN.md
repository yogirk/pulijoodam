Create `src/engine/ai/mcts.ts` with:

```typescript
export function mctsSearch(
  state: GameState,
  maxSims: number,
  timeBudgetMs: number,
  role: Role
): Move
```

Implementation:
- **MCTSNode** class/interface: `{ state, move (that led here), parent, children, visits, value, untriedMoves }`
- **Selection**: Walk tree using UCB1 (C=1.4) to find a leaf node with untried moves
- **Expansion**: Pick one untried move, create child node
- **Rollout**: Heuristic rollout (not pure random). Use `weightedRandomMove`:
  - For tiger: weight captures 5x, moves toward goats 2x, other moves 1x
  - For goat: weight moves adjacent to tigers 3x (blocking), clustering moves 2x, other 1x
  - Max rollout depth: 60 moves (then use evaluate() for non-terminal)
- **Backpropagation**: Propagate normalized score back up the tree. Normalize evaluate() output to [0,1] range for MCTS value averaging.
- **Time budget**: Check `performance.now()` every 100 iterations. Stop early if time exceeded.
- **Simulation cap**: Also stop at maxSims.
- **Return**: From root children, select child with most visits. If `topN > 1`, pick randomly from top N by visit count.

Handle chain-hop correctly: during rollout, if `chainJumpInProgress !== null`, only CAPTURE and END_CHAIN are legal. Do NOT switch perspective mid-chain. Treat the entire chain sequence as one "turn" in the tree.

Create test file covering the behavior cases. Tests use small sim counts (50-200) for speed. Construct specific states using createGame() + applyMove sequences.

**Minimax** (`src/engine/ai/minimax.ts`):

```typescript
export function minimaxSearch(
  state: GameState,
  maxDepth: number,
  timeBudgetMs: number,
  role: Role
): Move
```

Implementation:
- **Negamax** variant with alpha-beta pruning
- **Iterative deepening**: Start at depth 1, increment to maxDepth. If time runs out mid-depth, return best move from previous completed depth.
- **TimeoutError**: Check `performance.now()` against deadline every 1000 nodes. Throw TimeoutError to abort current depth.
- **Move ordering** (critical for alpha-beta efficiency):
  1. Transposition table best move (if exists)
  2. Captures (sorted by eval gain)
  3. Non-captures (sorted by lightweight eval heuristic)
- **Transposition table**: Fixed-size array (2^18 = 262144 entries). Store: `{ hash, depth, score, flag: EXACT|LOWER|UPPER, bestMove }`. Replace-always strategy. Clear between top-level calls.
- **Chain-hop handling**: When `chainJumpInProgress !== null`, only generate CAPTURE and END_CHAIN moves. Do NOT decrement depth for chain continuations (they're part of the same turn).
- Handle `topN` from AIConfig: if topN > 1, collect root-level scores and pick randomly from top N (for Easy difficulty randomization).

**Update chooseMove** (`src/engine/ai/index.ts`):

  - `state.phase === 'movement'` → `minimaxSearch(state, config.minimaxDepth, config.timeBudgetMs, state.currentTurn)`
  - Edge case: if `chainJumpInProgress !== null` during movement, still use minimax (chain is mid-turn, phase is movement)
